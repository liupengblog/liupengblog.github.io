Java Stream--(2)常用操作

发表于 2019-02-24 | 分类于 ModernJava

Stream和其它集合类的区别在于：其它集合类主要关注与有限数量的数据的访问和有效管理(增删改)，而Stream并没有提供访问和管理元素的方式，而是通过声明数据源的方式，利用可计算的操作在数据源上执行，当然BaseStream.iterator() 和 BaseStream.spliterator()操作提供了遍历元素的方法。

Java Stream提供了提供了串行和并行两种类型的流，保持一致的接口，提供函数式编程方式，以管道方式提供中间操作和最终执行操作，为Java语言的集合提供了现代语言提供的类似的高阶函数操作，简化和提高了Java集合的功能。

Spring boot – CommandLineRunner接口

Spring boot的CommandLineRunner接口用于执行一段代码–在应用程序初始化后执行，并且在应用的整个生命周期中仅仅被执行一次。

一. 如何使用CommandLineRunner

三种方式:

将CommandLineRunner作为一个@Component

最简单的一种.

将CommandLineRunner作为一个Bean

可以在SpringBootApplication中定义一个Bean，让这个Bean返回一个实现了CommandLineRunner接口的类.

ApplicationStartupRunner.java

注册ApplicationStartupRunner Bean

二. 多个CommandLineRunner接口实现时使用@Order

可以在一个应用定义多个CommandLineRunner接口的实现。默认情况下，spring boot会扫描所有的run方法并执行。你可以使用@Order注解来定义这些CommandLineRunner实现的执行顺序。

确认log结果如下(删了类名信息):

可以看出@Order值越小，对应的CommandLineRunner实现类就越早执行。

三. 为何要使用CommandLineRunner?

  • 如果需要在应用程序启动后仅仅执行一次的业务代码，CommandLineRunner就是一种有用的技术手段;
  • 顺便告诉你，Spring Batch 就是依赖CommandLineRunner来触发各种job的执行;
  • 利用command-line runner的这个特性，再配合依赖注入，可以在应用程序启动时后首先引入一些依赖bean，例如data source、rpc服务或者其他模块等等，这些对象的初始化可以放在run方法中。
