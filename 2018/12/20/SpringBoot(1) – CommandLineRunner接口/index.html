Spring boot – CommandLineRunner 接口

Spring boot 的 CommandLineRunner 接口用于执行一段代码–在应用程序初始化后执行，并且在应用的整个生命周期中仅仅被执行一次。

一. 如何使用 CommandLineRunner

二. 多个 CommandLineRunner 接口实现时使用@Order

可以在一个应用定义多个 CommandLineRunner 接口的实现。默认情况下，spring boot 会扫描所有的 run 方法并执行。你可以使用@Order 注解来定义这些 CommandLineRunner 实现的执行顺序。

确认 log 结果如下(删了类名信息):

可以看出@Order 值越小，对应的 CommandLineRunner 实现类就越早执行。

三. 为何要使用 CommandLineRunner?

  • 如果需要在应用程序启动后仅仅执行一次的业务代码，CommandLineRunner 就是一种有用的技术手段;
  • 顺便告诉你，Spring Batch 就是依赖 CommandLineRunner 来触发各种 job 的执行;
  • 利用 command-line runner 的这个特性，再配合依赖注入，可以在应用程序启动时后首先引入一些依赖 bean，例如 data source、rpc 服务或者其他模块等等，这些对象的初始化可以放在 run 方法中。
