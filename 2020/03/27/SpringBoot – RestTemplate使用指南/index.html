常用接口举例

Get 请求

有以下几种方式获取服务端 GET 请求的结果：

  • getForObject(url, classType) – 获取指定 url 的 GET 请求结果，并将结果转换为指定类型的对象.
  • getForEntity(url, responseType) – 获取指定 url 的 GET 请求结果，并将结果转换为通用 Http 响应对象.
  • exchange(requestEntity, responseType) – 执行指定的请求对象，返回 ResponseEntity.
  • execute(url, httpMethod, requestCallback, responseExtractor) – 按照指定的 Http 方法访问 URI 请求, requestCallback 用于准备请求对象, ResponseExtractor 用于读取响应结果.

Head 请求

Head 请求用于获取响应头信息,不是太常用,在个别的接口中(比如华为云的部分接口)很多有用的信息会展现在响应头中,这时 Head 请求就派上用场了。
由于不太常用,我们这里只举一个最简单的例子。

Post 请求

有以下几种方式获取服务端 POST 请求的结果：

  • postForObject(url, request, classType) – 提交请求(request)，并期待获取指定类型(classType)的响应结果.
  • postForEntity(url, request, responseType) – 提交请求(request)，并期待获取 ResponseEntity 的响应结果.
  • postForLocation(url, request) – 提交请求(request)，并期待获取 Location 对象.
  • exchange(requestEntity, responseType)
  • execute(url, httpMethod, requestCallback, responseExtractor).
