Java中一条语句搞定集合类初始化

发表于 2020-05-01 | 分类于 JavaSE

出于各种理由，我们有时候需要(或者仅仅是就想这么做)使用一条表达式语句完成集合的初始化。
集合的初始化大同小异，使用最多的还是List，我们这里主要以 List 举例。

从数组中创建 List

由于数组能够进行文字量初始化，我们可以先初始化数组，然后将数组转为 List。

我们还可以使用可变参数机制让代码更加简洁。

这个地方有个坑一定要注意:

  • Arrays.asList 返回的类型为 Arrays.ArrayList, 并不是我们常用的 java.util.ArrayList。
  • Arrays.asList 返回的实例是定长的，所以往里面加元素是要报错的。

尽管有以上两个限制，但是初始化后列表还是java.util.List这个接口的对象，接口有的方法都是可以使用的，所以上述限制也不是致命的。

从 Stream 中创建 List(Java8)

Java8 的出现，让我们可以很容易将 Stream 转为各种集合类。使用 Stream 的工厂方法可以初始化集合元素。

使用工厂方法(Java9)

在 Java9 中，集合类增加了几个方便的工厂方法，我们可以拿来初始化集合元素。不过前提是你的 JDK 已经升到了 Java9…😏

这里也有坑 😎
上述工厂方法返回的实例是不可变的,要是往里加元素，是会抛出 java.lang.UnsupportedOperationException 异常的。

使用匿名内部类(双大括号初始化)

Java8 之前，这种方式广为使用，以前我就经常这么用 😜。

这种方式由于历史悠久，使用的人不算少，值得说一说。

解释

“双大括号初始化”这个说法有误导性，Java 中没有双大括号语法；上述方法本质上是匿名内部类实现方式。第一层大括号表示匿名内部类定义，我们的例子就是定义了一个ArrayList的匿名子类，第二层大括号表示类中静态初始化代码。详细的阐述不展开。

双大括号初始化优点

  • 相比创建后再初始化的传统代码相比，代码量少一点
  • 代码更可读(对部分人来说……)
  • 一个表达式语句就完成了列表的创建与初始化

双大括号初始化缺点

  • 这种初始化方法不是人人都知道
  • 每次都多创建了一个类
  • 不支持钻石操作符(<>)
  • 对于 final 类不支持(这是当然的，因为 final 修饰的类不能被继承)
  • 对于内部类实例有隐式的引用，有内存泄漏的潜在风险

总之，Java8 之后，已经不推荐使用这种初始化方式。
